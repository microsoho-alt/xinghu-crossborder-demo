# Xinghu diagnosis decision experience v2

Authority: `GRIP-ODV1-UXDEPTH-20260715` / Issue #21 comment `4977145716`  
Backend contract: `diagnosis-decision-contract-v2.0.0`

## Experience promise

A non-expert should answer within 30 seconds: proceed, conditional or pause; the three reasons; three opportunities and risks; the variable that can change the decision; and three thresholded next actions.

The visual direction is a restrained Xinghu decision instrument: warm paper surface, dark ink, evidence green, condition amber and blocking red. It uses hierarchy and whitespace instead of decorative charts. Each component states one conclusion and one action. Provider names and raw output never appear.

## Guided input

The single form keeps every v5 field and sample, grouped as six numbered steps: basics; customer and goal; price/cost/logistics; compliance and attributes; market/channel and missing evidence; review and confirm. The header shows completion and explains why missing costs, demand proof, certification and logistics evidence affect confidence. Image suggestions remain opt-in and never overwrite a user correction.

## Result layers

1. The decision cockpit is first and expanded: state, reason, confidence/missing warning, three opportunities, three blocking risks, influential variable/threshold, and three pass/fail actions. Its primary actions are correct inputs, compare scenario and inspect evidence.
2. Exploration follows: twelve expandable decision-module cards and the scenario laboratory. Every module shows the complete backend contract. Scenario changes show baseline versus alternative profit, risk, decision and affected dimensions with causal reasons.
3. The full v5 professional report is one clearly named collapsed region. It retains execution summary, product profile, financial table, seven dimensions, platforms, risks, OPC tasks, seven-day path, download and fused claims.

## Responsive and accessibility acceptance

- Desktop target: 1440×1000; mobile target: 390×844.
- No horizontal document overflow. Dense tables live only in collapsed professional detail and remain horizontally contained.
- Decision state is not communicated by color alone. Red is only blocking, amber only conditional, green only evidence-backed opportunity.
- Labels remain legible at mobile width; actions are full-width where needed; scenario comparison becomes a vertical baseline/alternative stack.
- Keyboard focus is visible, expanders have descriptive labels, and major values include plain-language meaning.

## Persistence acceptance

Text and image flows, suggestion confirmation, user corrections, progress, result reload, Markdown report download, report feedback and outcome feedback must all remain functional. A scenario is local until the user explicitly saves it; it cannot mutate the authoritative baseline report.
